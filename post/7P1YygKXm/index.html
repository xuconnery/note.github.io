黑苹果macOS升级OC文件config.plist更改备忘

2022-03-18 Views 技术414字2 min read繁

EFI底本文件：https://github.com/GeQ1an/MSI-B360M-MORTAR-HACKINTOSH-OPENCORE-EFI

Misc—Boot

PickerMode: 置为 External （填Builtin就是不使用任何主题，填External是UI主题）

ShowPicker: 置为True （显示开机启动盘选项，比如MAC,WINDOWS那些）

Timeout: 倒计时进入指定硬盘，这里我们按需求填写，我填写10，代表10秒钟进入指定硬盘

HideAuxiliary：置为 False （开机选项显示清楚NVRAM）

MISC—Security

AllowSetDefault：True （允许使用 CTRL + 回车 和 CTRL + 数字 锁定默认启动项）

OpenCore启动项失效，自动添加Windows boot manager？

方法一：未测试

不知道楼主WIN10的引导文件是放在哪里的，我以前是这样搞的
EFI下有CLOVER和MICROSOFT，我把Microsoft改成了Microsoft1，然后clover里面添加win10引导项
原因是主板会自动寻找GPT盘的EFI／Microsoft／Boot／bootmgfw.efi为默认第一引导。。只要你不存在这个玩意，就不会自动添加了（https://bbs.pcbeta.com/viewthread-1878424-1-1.html）

方法二：似乎可行

系统偏好设置-启动磁盘，手动确认下macOS默认启动

U盘测试引导

最好准备一个U盘大小无所谓。格式化为FAT32格式，将升级后的EFI文件夹，整体放入U盘的根目录下。
开机按F8/11/12（各品牌主板/机器），选择UEFI-U盘引导系统。进行测试。测试正常。没有问题后，在用此EFI替换硬盘的EFI。如果有问题，再调整测试。

大跨度版本升级，无法进入系统：

  • 重置一下 NVRAM 试试
  • 不行的话，将 Misc > Security > SecureBootModel 设置为 Disabled
